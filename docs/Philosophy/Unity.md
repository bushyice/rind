Unity is easy to misconceive. Most systems often seem unified from a distance as they share the same hardware, same kernel, same resources-

Afterall, we see a computer as one-unit.

## Fragmentation

We are used to fragmentation, as daily, we encounter separate facets of software we touch. Looks feel different- layers of software feel separated as if they were not meant for each other. Every application is its own world, every subsystem declares a language. [[Communication]] becomes translation and [[Continuity|continuity]] becomes reconstruction.

Fragmentation separates understanding.

## Uniformity

Enforcement of true equilibrium. A brittle law of machine layers to be under one centralized actor, one language, one philosophy. one architecture.

An extreme solution to fragmentation by removing difference. Every layer is expected to conform to the same shape until coherence is achieved. 

Uniformity enforces understanding.  

## Unity

[[Communication|Communicating]], [[Communication#Understanding|understanding]] and [[Reactivity|reacting]] in unity all in one [[Continuity|continuous]] shared reality. That is true unity- It's far from fragmentation and against uniformity.
  
Unity allows understanding.

Unity is a **choice**, as it can not be enforced, it can only be suggested.
